# 6.2. Project Management & Collaboration Tools

The team relied on a focused toolchain to coordinate work and keep everyone
aligned.

- **Git & GitHub.** The single source of truth for code, history, and
  collaboration; pull requests provided a place for review and discussion.
- **GitHub Actions.** Continuous integration and deployment, so every merge was
  automatically built and shipped (see [7.3](./spd_devops.md)).
- **Figma.** Used in the design phase for high-fidelity wireframes, the
  light/dark palette, typography standards, and responsive grid layouts.
- **Docker & Docker Compose.** Gave every team member a consistent local
  environment, removing "works on my machine" friction.
- **mdBook.** Used to author this progress book, which doubles as living project
  documentation.

The combination of a shared repository, automated pipelines, and a common design
source kept a multi-disciplinary team — developers, designers, QA, and writers —
working from the same picture.
