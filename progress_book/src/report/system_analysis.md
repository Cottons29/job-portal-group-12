# 3. System Analysis & Requirements

This section translates the project's objectives into concrete requirements. It
is organized into **functional requirements** — what the system must do, grouped
by domain — and **non-functional requirements** — the qualities the system must
exhibit, such as performance, scalability, security, and accessibility. It closes
with the **success metrics** used to verify that those requirements were met.

The functional requirements are grouped into five domains that mirror the
system's modular structure:

| Domain | Responsibility |
| :--- | :--- |
| Identity & Social Graph | Accounts, authentication, profiles, follows |
| Community Content Engine | Posts, stories, likes, comments, shares |
| Intelligent Discovery Hub | Job search, filtering, skill-match recommendations |
| Collaborative Planning | Applications and the recruitment pipeline |
| Real-Time Interaction | Direct messaging and live notifications |

Each domain is detailed in the subsections that follow.
